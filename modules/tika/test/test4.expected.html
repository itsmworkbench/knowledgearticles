Elastic Search

Context

This is a ‘technical test’ to allow you to demonstrate your elastic search skills. We want you (on your own box, or in elastic search cloud as you choose) demonstrate the following. We will have a video meeting with screen share and look over the work at the end.

Timescales

We will have a meeting on Wednesday 8th to review the results

Data feed

  • We have a data feed from a Github organisation.
  • The organisation has members
  • The organisation has repositories
  • Load some of the files from this repository for searching (you can pick… you can even do it manually if you want, but scripting/code is far better)
  • We want to set up document level security for this

    • A suitable organisation is this https :// github . com / run - book or you can make your own, or use any other. The sample organisation only has two members.

    Goals:

  • We want to use the document level security as per https :// www . elastic . co / guide / en / elasticsearch / reference / current / field - and - document - access - control . html
  • Only people from the organisation and members of a specific group that you create can access their documents in search
  • I don’t want heavy configuration: this should be mostly a data exercise
  • While I don’t care about the importing of data, it’s important that the organisation data (who works for that organisation) is automated in some way

    • Not goals

  • We don’t care about the authentication mechanism. Use whatever you want
